# Build frontend app shell, protected routes, and login flow

## Description

Build the initial frontend shell from the prototype structure, including login, protected routing, navigation, and authenticated redirects.

## Acceptance Criteria

- [x] The login page submits credentials to `POST /api/session`.
- [x] Invalid credentials show a visible error state.
- [x] Authenticated users are redirected to Overview.
- [x] Unauthenticated users are redirected to Login when opening protected pages.
- [x] The shell includes navigation for Overview, Repositories, and Settings.
- [x] Typecheck/lint passes.
- [x] Verify in browser using dev-browser skill.
